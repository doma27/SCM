Software Problems

 

 

Fault

Possible Cause

 

1.             

Main SCM PC program will not boot.

Missing or corrupt files.

Refer to the SarcladSCMboot.LOG file in the root directory for an explanation.

Main SCM folder missing or incorrect name.

Check that the folder SCM.041 exists in C:\ directory and that all sub directories are present.

2.             

Data transfer and calibration buttons missing from the main PC screen.

Communications port is unavailable so the program has booted into viewer only mode.

Close other windows programs that may be using the com 1 port.

Uninstall any recently installed third party programs. 

3.             

Data will not print.

Default printer unavailable.

Change/set default printer in windows printer settings.

4.             

Downloaded data not available for viewing.

Failed Analysis.

Check gap and inclinometer Devices that measure angles relative to the horizontal sensor for correct operation. Replace or recalibrate as required.

SCM has not passed through curve section of caster.

Measurement performed too quickly.

Speed of measurement should not exceed 1.5 meters / min.

Multiple changes of direction.

Repeat measurement with limited direction changes.

5.             

Client PC stalls when booting or viewing data.

Connection to server PC has been lost.

Check that server PC is on and is not locked up.

Check that network is functioning.

6.             

Cannot connect to SCM using Wi-Fi

Customer laptop may not be connected to the Moxa router

Check laptop to make sure it automatically connects to the Sarclad Moxa router in Wi-Fi settings.

Moxa unit may require reboot

Turn off the power to the Moxa Wi-Fi Router and re-connect with laptop.
